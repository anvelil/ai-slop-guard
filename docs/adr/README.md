# Architecture Decision Records

Each ADR captures one decision, why it was made, and what it costs. Read
these before proposing a change that contradicts one — either the ADR's
reasoning still holds and the proposal needs to account for it, or the
context has changed and the ADR should be superseded (add a new one, mark
the old one's status as "Superseded by NNNN", don't delete it).
